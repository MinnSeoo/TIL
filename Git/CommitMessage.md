# **좋은 Git commit 메세지를 작성하는 방법**

## **Why ???**
&nbsp; Git commit 메세지를 **왜** 잘 써야 할까???<br>
- 더 좋은 커밋 로그 가독성.
- 더 나은 협업과 리뷰 프로세스.
- 더 쉬운 코드 유지보수. <br>

&nbsp; 대표적으로 3가지의 이유때문이다.<br>

<br>

## **좋은 Git commit 메세지를 작성하기 위한 약속**

- 되도록 영어로 작성하기.
- 제목과 본문을 한 줄 띄어 분리하기.
- 제목은 50자 이내로 작성 (영어 기준).
- 제목 끝에 ```.``` 금지.
- 제목은 **명령조** 로.
